# 项目进度

更新时间：2026-04-06

## 当前总体判断

当前仓库仍处于“前期基础搭建 / 数据接入 MVP”阶段，还不是 `tasks.md` 里规划的完整毕业设计成品。

已落地的核心内容：
- Python 统一环境方案已经建立，入口是根目录 `pyproject.toml` + `.venv` + `uv`。
- 后端已完成一批“数据源与数据落盘”能力：AkShare 股票日线抓取、Olist 数据集校验、合成电商数据生成、演示站爬虫、manifest 契约。
- `backend/tests` 里已有 2 个测试文件、3 个测试用例，当前可以通过。
- `data/processed` 里已经有 synthetic smoke 数据和测试产物，说明“生成 CSV + 写 manifest”链路已跑通。

尚未落地的主线内容：
- Vue3 前端工程基本未开始。
- FastAPI 的 API 分层、数据库模型、schemas、services、repositories、鉴权、迁移脚本均未开始。
- C++ 算法引擎、PyBind11 绑定、性能实验尚未开始。
- Docker Compose 部署、Nginx、正式论文/设计/API/实验文档尚未开始。

已验证状态：
- `uv run python backend/scripts/check_environment.py` 可运行，核心命令与核心 Python 包均可用；可选包 `tushare`、`scrapy` 仍缺失。
- `uv run python -m unittest discover backend/tests -v` 已通过 3 个测试。

## 对“实际已完成内容”的补充说明

虽然 `tasks.md` 规划的是完整的前后端 + 算法引擎系统，但当前仓库真正已经完成的内容，主要集中在下面这些点：

1. 根目录工程化基础
   - `pyproject.toml` 已成为 Python 依赖主入口。
   - `.env.template` 已提供运行时配置模板。
   - `backend/requirements.txt`、`backend/requirements-optional.txt` 已能从 `pyproject.toml` 自动导出。

2. 后端已落地的数据源能力
   - `backend/app/data_sources/contracts.py`：定义了统一 `SourceType`、`ImportManifest`、`DatasetArtifact` 契约。
   - `backend/app/data_sources/stock.py`：已实现 AkShare A 股日线抓取、字段标准化、CSV 与 manifest 输出。
   - `backend/app/data_sources/olist.py`：已实现 Olist 数据集结构校验和 manifest 输出。
   - `backend/app/data_sources/ecommerce_synthetic.py`：已实现多表电商模拟数据生成。
   - `backend/app/data_sources/demo_crawler.py`：已实现演示电商站爬取与结果落盘。

3. 命令行脚本层
   - 已有环境检查、requirements 导出、股票抓取、Olist 校验、电商合成数据生成、演示爬虫 6 个脚本。
   - 这说明“脚本驱动的数据接入 MVP”已经可用。

4. 数据与测试
   - `data/processed/` 里已有 synthetic smoke 数据与测试产物。
   - 后端测试当前覆盖 manifest 序列化和合成数据引用一致性，属于基础正确性验证，覆盖面还比较有限。

## 一句话结论

如果按毕业设计完整系统来衡量，当前进度更接近“已完成基础环境、数据源接入 MVP 和少量测试”，还没有进入“完整前后端联调 + 数据库建模 + 算法引擎集成 + 部署与实验报告”的阶段。
