# 毕业设计方案：基于高级数据结构的高性能金融时序数据分析引擎与管理系统

## 一、 系统核心定位与架构

本项目采用**微服务/引擎化架构**，告别传统的纯业务增删改查（CRUD），从零构建一个轻量级的时序数据库引擎，以满足极高的工作量要求并展现深厚的技术壁垒。

* **底层核心计算与存储引擎 (C++)：** 负责将海量股票/金融时序数据加载到内存，并维护主席树、线段树等复杂数据结构，提供超高性能的 RPC 查询接口。
* **后端业务与数据网关 (Python)：** 负责对接前端的 HTTP 请求，处理用户鉴权、数据爬取与清洗，并通过 gRPC 与 C++ 核心引擎通信。
* **前端交互与可视化 (Vue3/React + ECharts)：** 提供金融数据看板、动态 K 线图展示、高频查询控制面板以及系统状态监控。

## 二、 核心算法与数据结构的应用落地

将 ACM 竞赛中的高级数据结构完美映射到实际的金融高频查询场景，这是本项目的核心创新点：

### 1. 线段树 (Segment Tree) —— K 线数据的动态区间聚合
* **业务场景：** 用户在前端缩放 K 线图时，实时查询任意时间区间 $[L, R]$ 内的最高价、最低价、成交量总和等。
* **技术实现：** 在 C++ 引擎中为高频交易数据构建线段树，在 $O(\log N)$ 的时间复杂度内返回任意区间的统计结果，并支持实时数据的单点/区间更新。

### 2. 主席树 / 可持久化线段树 (Persistent Segment Tree) —— 历史版本与多维排名查询
* **业务场景：** 金融回测系统中的“时间穿越”查询，例如：“获取某历史月份中，单日成交量排名第 $K$ 大的股票”，或查询特定历史版本的交易分布。
* **技术实现：** 利用主席树保留所有历史版本的数据状态。结合权值主席树解决静态/动态区间第 $K$ 大问题，实现对海量历史数据的高效排名与筛选。

### 3. 分块 / 根号算法 (Square Root Decomposition) —— 复杂规则的区间更新
* **业务场景：** 处理股票市场的“复权”操作，或对特定板块股票施加非线性的修正因子（不满足结合律，无法使用线段树的 Lazy Tag）。
* **技术实现：** 将时间序列或股票池分为 $\sqrt{N}$ 块。对整块进行打标记处理，对散块进行暴力重构，确保在 $O(\sqrt{N})$ 的复杂度下完成极度复杂的动态修改。

### 4. CDQ 分治 / 多维偏序 —— 复杂多因子选股策略
* **业务场景：** 实现量化交易中的多因子选股。例如筛选满足：市盈率 $PE < A$，且换手率 $Turnover > B$，且收益率 $Yield > C$ 的股票集合。
* **技术实现：** 将选股问题转化为三维或多维空间中的偏序问题。在 C++ 层使用 CDQ 分治配合树状数组 (Fenwick Tree)，实现 $O(N \log^2 N)$ 的高效离线降维筛选。

## 三、 模块与语言职责划分

### 模块 1：C++ 高性能时序引擎 (`CoreEngine`)
* **核心职责：** 数据加载、内存管理、底层数据结构维护与查询计算。
* **技术栈：** C++17/20, gRPC, Protobuf, 线程池。
* **工作内容：** * 构建自定义内存结构以缓存 CSV 或实时流数据。
    * 封装上述四种核心数据结构。
    * 暴露 gRPC 服务接口（如 `GetRangeStats`, `GetKthVolume`, `FilterByFactors`）。

### 模块 2：Python 业务与数据管道 (`DataGateway`)
* **核心职责：** 数据获取、API 路由转发、基础业务逻辑。
* **技术栈：** Python 3.10+, FastAPI, Akshare (金融数据接口), Pandas, SQLAlchemy。
* **工作内容：** * 编写定时任务（如 Celery），每日自动拉取最新的股票日线/分钟线数据。
    * 使用 Pandas 进行数据清洗与预处理。
    * 处理用户账户系统的 CRUD 逻辑（对接 MySQL/SQLite）。

### 模块 3：现代化前端 UI (`WebDashboard`)
* **核心职责：** 数据交互、可视化图表渲染。
* **技术栈：** Vue 3, TypeScript, Element-Plus, ECharts。
* **工作内容：** * 设计暗黑模式的专业级金融交易看板。
    * 集成 ECharts 绘制可交互、可缩放的专业 K 线图。
    * 开发“因子选股器”面板，实时对接后端的多维偏序筛选接口。

## 四、 项目推进路线图 (Roadmap)

* **阶段一：基础设施与数据准备 (2 周)**
    * 完成项目整体架构搭建与 Protobuf 协议定义。
    * 使用 Python 爬取并清洗至少 10 年期的 A 股/美股核心股票历史数据。
* **阶段二：C++ 核心引擎攻坚 (4 - 5 周)**
    * 在 C++ 中设计并实现基础内存存储结构。
    * 依次开发线段树、主席树、分块、CDQ 分治模块，并完成严格的单元测试（确保无内存泄漏）。
    * 搭建并跑通 gRPC Server。
* **阶段三：Web 后端与前端开发 (3 周)**
    * 开发 FastAPI 接口网关。
    * 完成 Vue3 前端页面开发与 ECharts 联调，实现全链路贯通。
* **阶段四：性能压测与论文撰写 (3 周)**
    * 使用压测工具（如 JMeter/Wrk），对比自定义 C++ 引擎与传统 MySQL/Redis 在千万级历史记录下的查询耗时。
    * 将压测数据可视化，作为论文最核心的“性能测试与分析”章节。

## 五、 项目亮点与答辩优势

1.  **拒绝平庸：** 摒弃传统的全套 Java/Python CRUD 模式，引入底层独立计算引擎架构，具备极强的系统级工程属性。
2.  **硬核落地：** 成功将 ACM 算法竞赛中抽象的高级数据结构转化为工业界高频交易查询的解决方案，完美展现算法功底与知识转化能力。
3.  **跨语言工程：** 涵盖 C++ 高性能计算、Python 数据处理与高并发网关、以及现代前端可视化，展现全面的全栈架构能力与微服务协同思维。
